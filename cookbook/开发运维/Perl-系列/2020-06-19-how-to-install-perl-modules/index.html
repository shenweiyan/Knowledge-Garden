    You should run the cpan command as your normal user. You have two choices:

    1. Install modules into a directory under your home dir. local::lib will help you set that up.
    2. Configure cpan to use sudo during the install phase. You do that by starting the cpan shell and typing:
    o conf make_install_make_command 'sudo make'
o conf mbuild_install_build_command 'sudo ./Build'
o conf commit

    The first line configures MakeMaker to use sudo. The second line does the same for Module::Build. The third line saves the changes.

    Make жҳҜдёҖз§ҚиҮӘеҠЁеҢ–жһ„е»әе·Ҙе…·пјҢз”ЁдәҺд»Һжәҗз ҒиҮӘеҠЁз”ҹжҲҗеҸҜжү§иЎҢзЁӢеәҸе’Ңеә“пјҲйҖҡеёёеҖҹеҠ©дәҺ makefile жқҘжҺЁеҜјеҮәзӣ®ж ҮзЁӢеәҸпјүпјӣMake жңүеӨҡдёӘзүҲжң¬пјҢеҰӮ Unix/Linux дёӢ зҡ„ GNU MakeпјҲд№ӢжүҖд»Ҙжңү gmakeпјҢжҳҜеӣ дёәеңЁеҲ«зҡ„е№іеҸ°дёҠ make дёҖиҲ¬иў«еҚ з”ЁпјҢGNU make еҸӘеҘҪеҸ« gmake дәҶпјүгҖҒWindows дёӢзҡ„ Microsoft nmake зӯүгҖӮdmake еҲҷжҳҜдёҖдёӘйҖӮз”ЁдәҺ LinuxгҖҒSolarisгҖҒWin32 д»ҘеҸҠе…¶д»–е№іеҸ°зҡ„и·Ёе№іеҸ°зҡ„е‘Ҫд»ӨиЎҢзј–иҜ‘е·Ҙе…·пјҢйҷӨжӯӨд№ӢеӨ–пјҢи·Ёе№іеҸ°зј–иҜ‘е·Ҙе…·дёӯиҝҳжңүдёҖдёӘеӨ§еҗҚйјҺйјҺзҡ„ cmakeгҖӮ

    е…ідәҺ Make зҡ„иҜҰз»Ҷд»Ӣз»ҚпјҢе…·дҪ“еҸҜеҸӮиҖғпјҡWikipedia: Make (Software)гҖӮ

    dmake е·Ҙе…·пјҡhttps://metacpan.org/release/dmakeпјҢе…·дҪ“е®үиЈ…ж–№жі•иҜ·иҮӘиЎҢи°·жӯҢпјҲdmake иІҢдјјжҳҜ cpan зҺҜеўғй…ҚзҪ®еҘҪе°ұжңүдәҶеңЁ “C:\Perl\site\bin” дёӢпјүгҖӮ

    nmake жҳҜ Microsoft Visual Studio дёӯзҡ„йҷ„еёҰе‘Ҫд»ӨпјҢйңҖиҰҒе®үиЈ… VSпјӣе®үиЈ… VS еҗҺпјҢnmake.exe еҮәзҺ°еңЁ В C:\Program Files (x86)\Microsoft Visual Studio 9\VC\binВ В  зӣ®еҪ•дёӢпјҲbin ж–Үд»¶еӨ№зҡ„дҪҚзҪ®ж №жҚ®жҜҸдёӘдәәзі»з»ҹдёҚеҗҢпјҢжүҫеҲ°иҮӘе·ұзҡ„е®үиЈ…дҪҚзҪ®пјҢbin ж–Үд»¶еӨ№дёӯеҢ…еҗ« nmake еҚіеҸҜпјүгҖӮеҰӮжһң VS е®үиЈ…иҝҮзЁӢдёӯпјҢжІЎжңүжіЁеҶҢ VS зҡ„зҺҜеўғеҸҳйҮҸпјҢеӣ жӯӨзӣҙжҺҘеңЁ cmd зҡ„ DOC дёӯиҫ“е…Ҙ nmake е°ұдјҡеҮәзҺ°пјҡ“‘nmake’ дёҚжҳҜеҶ…йғЁжҲ–иҖ…еӨ–йғЁе‘Ҫд»ӨпјҢд№ҹдёҚжҳҜеҸҜиҝҗиЎҢзҡ„зЁӢеәҸжҲ–жү№еӨ„зҗҶж–Үд»¶”В  жҠҘй”ҷгҖӮ
